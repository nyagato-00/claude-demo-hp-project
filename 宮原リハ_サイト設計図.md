## 0. この文書の使い方

- **役割**：作るものの全体像（情報設計・部品・コピー方針・実装の対応）を 1 枚にまとめた「地図」。
- **ダミー前提**：宮原リハは**架空の事業所**です。法人名・住所・電話・氏名はすべてダミー。実在の個人情報は載せません（→ 1-6 セキュリティ）。
- **更新方針**：料金・プラン・UI など**変わりやすい数値は焼き込まない**。考え方と構成を固定し、具体は本番で都度確認します。

## 2. サイトの目的とターゲット

- **このサイトの役割＝「届ける」**：宮原リハが*どんな事業所か*を伝える HP。データを「ためる」アプリではない（→ 5-1 静的＝届ける／ためる・アプリは 3 章・6 章）。
- **読む人（3 者）**
  - **ご本人・ご家族**：安心して任せられそうか、何をしてくれるのかを知りたい。
  - **ケアマネジャー**：対応エリア・サービス内容・連絡先をすばやく確認したい。
  - **退院支援の医療職**：紹介先として概要を把握したい。
- **サイトのゴール（1 つに絞る）**：**「お問い合わせ・相談」につなげる**こと。各ページの導線は最終的にここへ向ける。

## 5. ページ別 設計（セクション構成＋コピーの方向性）

### 5-1. トップページ
| セクション | 内容 | コピーの方向性 |
|---|---|---|
| ヒーロー | キャッチ＋一言説明＋相談ボタン | 「ご自宅でのリハビリを支えます」など*事実ベース*の安心感 |
| 強み 3 つ | 例：①多職種(PT/OT/ST) ②生活に沿った支援 ③医療との連携 | 断定や“No.1”を避け、できることを具体的に |
| 対象の方 | こんな方へ（退院後・在宅生活の不安 等） | 共感を煽りすぎない、落ち着いたトーン |
| ご利用の流れ（要約） | 相談→契約→開始の 3 ステップ | 詳細ページへ誘導 |
| お問い合わせ導線 | フォーム/電話へ | ゴールへ集約 |

### 5-2. サービス内容
- 訪問リハビリとは（在宅で受けられるリハの説明）
- PT / OT / ST それぞれの役割（生活場面に即して）
- 訪問でできること・できないことの整理

### 5-3. ご利用の流れ・対象
- ステップ：①相談 → ②ケアマネ/主治医と調整 → ③契約 → ④訪問開始
- 対象となる方の例（保険区分の前提に軽く触れる）

### 5-4. 対応エリア・料金の目安
- 対応エリア（伊東市中心・地図 or テキスト）
- **料金は具体額を焼き込まない**：介護保険／医療保険の区分・自己負担の*考え方*を案内し、「詳細はお問い合わせください」へ。

### 5-5. スタッフ紹介
- 職種・人数・体制（顔写真は任意。出す場合もダミー・本人同意前提）

### 5-6. アクセス・事業所情報
- 住所・電話・営業時間・地図（埋め込みは任意）

### 5-7. お問い合わせ
- フォーム（外部）または電話。**静的サイトはデータを保存できない**ため、3 章のフォーム（Tally / Google フォーム）やメールにつなぐ（→ 第 10 節）。

## 6. コピーライティングのガードレール（重要）

> 訪問リハの発信は、**医療広告・薬機法・景品表示法**に関わります。note-writer の方針と揃えます。
> ※これは一般的な注意点であり、法的助言ではありません。適用される規制は**事業形態（医療機関か介護事業所か）で異なる**ため、最終的な表現は専門家・自治体に確認してください。詳細は **5-6** で扱います。

### 6-1. 避けたい表現
- **断定・保証**：「必ず良くなる」「治る」「改善します」→ 効果の保証は不可。
- **体験談で効果を示す**：患者・家族の「歩けるようになった」等の*効果に関する体験談*は、ウェブでも原則 NG。
- **ビフォーアフター**：効果を誤認させる前後比較は、所定の説明要件を満たさない形では不可。
- **比較優良**：「地域 No.1」「最高の」など、根拠なく他と比べて優れているとする表現。
- **誇大・キャンペーン強調**：事実を不当に大きく見せる／「期間限定」等で過度に誘引する見せ方。

### 6-2. OK な書き方（言い換えの方向）
| NG（避ける） | OK（事実ベース） |
|---|---|
| 必ず歩けるようになります | お一人おひとりの状態に合わせ、生活動作の維持・向上を目指します |
| 地域で一番の実績 | PT・OT・ST が在籍し、多職種で支援しています |
| 利用者さんの「劇的に改善した」の声 | 提供しているサービスの内容を具体的に説明（体験談に頼らない） |
| 今だけ無料キャンペーン！ | まずはお気軽にご相談ください（過度な強調はしない） |

### 6-3. 運用ルール
- Claude にコピーを書いてもらうときも、**この方針を一緒に伝える**（「医療広告ガイドラインに配慮し、断定・体験談・比較優良を避けて」）。
- 公開前に 6-1 のチェックリストで目視確認。

## 7. デザイン方針（トーン＆マナー）

- **印象**：清潔・安心・親しみ。医療/介護らしい落ち着き。
- **配色（たたき台・調整可）**

| 役割 | 色（例） | 用途 |
|---|---|---|
| ベース | 白 + 余白多め | 背景 |
| メイン | 落ち着いた青緑（例 `#2A7F8E`） | 見出し・ヘッダー |
| アクセント | 温かみのあるオレンジ（例 `#E8833A`） | CTA ボタン |
| 文字 | 濃いグレー（例 `#333333`） | 本文 |

- **可読性（高齢者・ご家族配慮）**：本文は**大きめ（16–18px 目安）**、行間ゆったり、文字と背景の**コントラストを確保**（WCAG AA 目安）。
- **レイアウト**：1 カラム中心でシンプルに。重要導線（相談・電話）は常に見える位置。
- **レスポンシブ**：スマホ閲覧が多い前提。指で押しやすいボタンサイズ。
- **画像**：人物・写真はダミー／フリー素材＋本人同意前提。容量に注意（→ 5-6 つまずき）。

## 8. Astro 実装マッピング（概念レベル）

> ファイル名・書き方は**変わることがある**ため、構造の考え方だけ示します（実際は動画・公式で確認）。

```
プロジェクト/
├─ src/
│  ├─ components/   … 共通部品（Header / Footer / Cta / FeatureCard …）
│  ├─ layouts/      … ページ共通の枠（ヘッダー＋本文＋フッター）
│  └─ pages/        … 1 ファイル = 1 ページ（トップ / サービス / アクセス …）
└─ public/          … 画像など（そのまま公開されるファイル）
```

- **components/** ＝ 第 4 節の「使い回す部品」。
- **layouts/** ＝ ヘッダー・フッターを 1 か所で持つ枠（共通の見た目）。
- **pages/** ＝ サイトマップ（第 3 節）の各ページに対応。本文だけ差し替える。

| サイトのページ | 対応する pages のイメージ |
|---|---|
| トップ | `index`（サイトの入口） |
| サービス内容 | `service` |
| ご利用の流れ | `flow` |
| 対応エリア | `area` |
| スタッフ紹介 | `staff` |
| アクセス | `access` |
| お問い合わせ | `contact` |

## 10. 「ためる」側へのつなぎ（お問い合わせ）

- 静的サイト（このサイト）は**データを保存できない**。問い合わせ内容を受け取るには外部の仕組みが要る。
- **選択肢**
  - **電話・メール（mailto）**：最も簡単。
  - **外部フォーム**：3 章で学んだ Tally / Google フォームを埋め込む or リンク。
- **個人情報の扱い**：氏名・連絡先など個人情報を受ける窓口になるため、収集・共有範囲に注意（→ 1-6）。フォーム側のプライバシー設定を確認。
